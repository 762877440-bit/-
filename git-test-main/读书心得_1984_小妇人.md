# 在电幕的冷光与阁楼的炉火之间——读《1984》与《小妇人》有感

## 序章：一本书让人坠入深渊，一本书让人浮出水面

这件事要从大二的寒假说起。

我把《1984》和《小妇人》放在同一个书包里带回老家，这个组合让室友感到困惑。他说，一本书是灰的，一本是粉的；一本让你怀疑人类，一本让你相信人类。你到底是去地狱还是去天堂？

我说，也许两者都是真实的。

整个寒假，我经历了一场奇异的阅读体验。白天窝在沙发里，看温斯顿·史密斯在电幕的监视下战战兢兢地写下日记，看大洋国的"新话"如何一点一点地削薄人的思想，看真理部的口号"战争即和平，自由即奴役，无知即力量"像三把锤子凿进阅读者的颅骨。每到傍晚，母亲喊我吃饭，我从书页上抬起头，需要用力摇晃一下脑袋，才能确认窗外的世界还是那个可以自由呼吸的世界。

而到了夜晚，我翻开《小妇人》，走进马奇家那间清贫却温暖的阁楼。乔伏在火炉边写作，贝丝弹着走了音的钢琴，梅格在和劳里跳舞，艾米在梦想着去巴黎学画。马奇太太坐在一旁，目光温柔得能让整个新英格兰的冬天融化。

我在这两本书之间来回摆荡，像一只被夹在风暴与暖流之间的鸟。

这不是一个轻松的阅读组合。它让我在同一个冬天里经历了人性的两极——一极冷到骨头里，一极暖到心窝里。而我作为一个学软件工程的学生，在这些黑白分明的文字之间，看到了一个灰色的问题，横亘在我未来的职业道路上：技术，你到底是要做电幕，还是要做炉火？

## 第一乐章：冰与火之歌——《1984》的极权深渊

大洋国的伦敦是灰的。

这个颜色不是写出来的，是从每一行文字里渗出来的。奥威尔没有用太多笔墨渲染环境的灰暗，但读完前十页，我的视觉记忆里只剩下了一种颜色——那种介于水泥和旧报纸之间的、让人透不过气的灰。

温斯顿住在胜利大厦的一间破屋里，厨房的窗户对着一个大院，院里有一台电幕。电幕是圆的，像一只钢化玻璃的眼球，嵌在每一面墙上——你家墙上、走廊里、地铁站、办公室，甚至厕所。电幕不说话的时候，它在听；它说话的时候，你只能听。它可以把音量调到足以覆盖任何私语，也可以精确地捕捉到你在楼道里嘟囔的那半句抱怨。

我在敲这段文字的时候在想，我的手机、我的笔记本电脑、我的智能音箱，它们是不是也像一个个迷你电幕？我不是在说阴谋论。我是说，当我在做我的课设——一个基于用户画像的推荐系统——的时候，我突然感到后背发凉。我当时坐在实验室里，屏幕上是一排排Python代码，有一个模块是我自己写的，用来分析用户点击行为。那一刻，我停下敲键盘的手，看了看屏幕右上角的摄像头，用一张便利贴把它贴住了。

这是《1984》给我的第一个后遗症：对"注视"变得敏感。

"老大哥在看着你。"这五个字出现在每一张海报上、每一枚硬币上、每一块牌子上。但真正可怕的不是有人在看你，而是你知道有人在看你，但你不知道他是谁。这种不确定性的恐惧是最高效的统治工具。它让你在开口之前先替沉默辩护，在动笔之前先把笔放下。

"新话"是奥威尔在这个世界里最令人脊背发凉的发明。它不只是一个语言压缩方案，更是一个思想阉割手术。如果词汇量每年都在缩减，那么那些无法被表达的复杂思想就会自然消亡。你无法谈论自由，因为这个词已经被删除了；你无法抗议不公，因为你的语言库中没有可以用来抗议的词汇少了。思想犯罪——thoughtcrime——是将思想本身变成一种可被惩罚的动作。在电幕面前，你没有隐私；在思想警察面前，你连内心都是透明的。

温斯顿最终被摧毁，不是因为他反抗了，而是因为他相信了爱。他和茱莉亚在乡村租了一间房，在远离电幕的地方，他们以为只要两个人在一起，人的完整性就不会被击碎。但如果一本书能让爱变成一种罪，那这本书描写的已经不是政治了，而是人性被连根拔起的全过程。

我记得合上这本书的那个午夜，我没有立刻入睡。我站在窗前，看对面公寓楼里零星的灯火。万家灯火这个词，在那天晚上失去了浪漫含义——我想的是，每一扇窗户后面也有一台可能的电幕吗？每一个亮着光的房间里，也有人在不敢说话吗？

我想多了。但一本真正好的书，会让你想多。

## 第二乐章：光与暖之诗——《小妇人》的人性花园

我不记得上一次因为一个虚构人物的幸福而感到幸福是什么时候了。但在读《小妇人》的某个深夜，马奇家的四个女孩围在炉火前讨论各自的梦想，我感到胸口一阵温热，像是自己也在那个房间里，坐在那张旧沙发上，腿上盖着一条打了补丁的毯子。

乔说她想写一本书。不是一本普通的书，是一本能让她挣钱养家的书。这个愿望朴素得让人心软，但她为此付出的代价——放弃写作去教书、放弃劳里、在阁楼上点着蜡烛写到凌晨——让我对"写作"这个词产生了敬意。乔是四个女儿中最不像"淑女"的一个。她莽撞、心直口快、不喜欢穿裙子、在舞会上踩掉自己的鞋。但她有一种罕见的诚诚实：对自己诚实，对感情诚实。当她拒绝劳里的求婚时，不是因为她不爱他，而是因为她知道自己不能以"妻子"的身份去爱他。这个决定，放在任何时代，都需要巨大的勇气。

我在敲这段文字的时候在想，如果马奇家生活在大洋国，乔的第一篇小说就会被举报。举报她的人是艾美，因为她们姐妹之间经常吵架吗？不，是因为在那个世界里，亲人之间也会互相举报，以保证自己不被牵连。而《小妇人》的世界恰恰相反——亲人之间的矛盾在温暖的晚餐桌上化解，争吵在炉火熄灭前结束。

马奇太太是最动人的人物之一。她独自一人撑起一个家，丈夫在战场上，她要养活四个女儿。但她从不抱怨。她说："我一生都在努力成为一个好母亲，就像我在学习成为一个好妻子一样。这个学习过程永远不会结束。"不是励志语录，不是座右铭，而是一个普通母亲在工作了一整天、烧好了晚饭、缝补了孩子们的裙子之后，说的最平常不过的话。

贝丝的病是全书最让人心碎的部分。这个安静、害羞、善良的女孩，因为帮助一个贫困的家庭而染上猩红热。在贝丝被病痛折磨的那些章节里，我可以清晰地感受到马奇家的温度——那种在一道门背后共同承受悲伤的力量。他们不会说"我们是家人，我们要坚强"这种话，他们只是轮流坐在贝丝的床边，给她念书、弹琴、掖被角。

我意识到，在这个家庭里，"注视"不是监控，而是关怀。马奇太太看女儿们的眼神里没有电幕的冷，只有炉火的暖。

如果让我用一种技术性地描述这本书，我会说它是一个关于"并行计算"的故事——四个线程（四个女儿）在共享内存（马奇家的温情）中各自运行，偶尔遭遇I/O中断（战争、疾病、爱情的挫折），但最终都顺利执行完毕，输出各自的人生结果。但这个比喻太技术了，太冷了。我更想说的是，马奇家让我相信，人在没有监控的情况下依然可以善良，人在可以自由选择的时候依然会选择爱。

## 第三乐章：技术的两副面孔——从电幕到炉火

我把这两本书放在一起读，不是因为我想要制造一个"黑暗vs光明"的二元对立。这个世界不是非黑即白的，技术也不是。

但有些对比，作为一个软件工程的学生，实在无法视而不见。

第一个对比是通讯工具。在《1984》里，电幕是单向的、强迫式的信息输入。你没有选择关掉它，你只能选择在它面前保持沉默。在《小妇人》里，马奇家最重要的通讯方式是书信。乔的母亲马奇太太在信中得知丈夫受伤，四个女儿挤在一起看信，哭声和叹息声像音符一样此起彼伏。电报、信件、日记——这些在《小妇人》里承载着情感的工具，到了《1984》里全部变成了监控和操控的手段。同一个技术基础，两种截然不同的结果。

区分它们的，不是技术本身，而是运用技术的人抱着怎样的目的。

第二个对比是语言。在《1984》里，新话在逐年缩小编典。少一个词，少一种表达，少一种反抗的可能。在《小妇人》里，乔的语言在扩张——她读书、写作、编故事，用文字描绘出一个属于自己的世界。她被退稿，但继续写；她的小说终于被出版，印成铅字的那一刻，我比她还激动。语言在《1984》中是囚笼，在《小妇人》中是翅膀。

而我的专业，软件工程，本质上就是一门语言+工具的学科。我们写的代码可以创造，也可以限制。一个API可以开放数据，也可以封锁数据。一个算法可以精准推荐一首你喜欢的歌，也可以精准推送一条让你愤怒的信息。代码本身没有道德，但代码写出来之后，它服务的目标有。

我这样说可能有些学生腔，但这确实是我在大二上学期学完《数据结构与算法》后的感受。我调试过一个递归函数，花了三个小时，最后发现是一个边界条件的错误。那一刻我感到的成就感，和我写出一段漂亮的排序算法时的愉悦，是真实的、纯粹的。但温斯顿的故事让我不安地想到，如果有一天我写的代码被用在某个我不愿意的地方呢？如果我的算法被用来压缩某人的语言、预测某人的行为、限制某人的选择呢？

我还没有找到一个让我安心的答案。

## 第四乐章：在代码与人性之间——一个软件工程学生的选择题

大二上学期，有一门课叫《软件工程导论》。教授在第一节课上说了一句话，我现在还记得："你写的每一行代码，最终都是在影响某一个人的生活。"

当时觉得这句话有点重。写了一个双端队列，它影响谁的生活了？但读完这两本书后，我重新理解了这句话的重量。

《1984》让我意识到，忽视伦理的技术是危险的。推荐系统可以变成思想灌输，社交网络可以变成舆论监控，人脸识别可以变成身份追踪。我不是在危言耸听，这些在今天的世界上正在发生。我们学软件工程的人，迟早要面对这样一个问题：如果老板让你写一个收集用户隐私的模块，你写不写？如果产品经理说"这个功能可以让竞争对手的数据消失"，你做不做？

温斯顿的选择是被迫的，但我们的选择是主动的。这是我和他最大的区别。

《小妇人》则从一个更温暖的角度给出了答案。乔为什么能写出让人感动的故事？因为她知道什么是重要的——她的家人、她的诚实、她的自由。马奇家不是与世无争的，她们面临战争、疾病、贫穷，但她们从未在困难中放弃自己坚守的东西。对于学计算机的人来说，这种"坚守"很抽象，但也很具体——知道自己不能跨越的边界在哪里，知道自己为什么要写代码，以及为谁写代码。

我在敲这段文字的时候在想，数据结构课上学到的"队列"和"栈"，如果放在马奇家会是什么角色？队列是轮流照顾贝丝的顺序，栈是马奇太太叠好的冬衣。课程里没有这种比喻，但生活里有。软件不是冷冰冰的逻辑，它是从人类的协作模式中抽象出来的。如果我们在学习抽象的时候忘记了人类协作的初衷，那我们学到的只是骨架，不是灵魂。

马奇家的故事让我相信另一件事：好的团队不在于成员有多强，而在于他们如何在困境中彼此扶持。这不正是敏捷开发中强调的"自组织团队"吗？乔有时候太冲动，但梅格总是温和地提醒她；艾美有时有些任性，但乔会在她受伤时第一个站出来保护她。这不是理想化的姐妹情，这是一个团队的自然状态——有摩擦，有谅解，有共同的目标，有各自的成长。

而《1984》里的"团队"是什么？是"友爱部"，一个以折磨人为目的的地方。那里没有协作的成果是摧毁一个人。奥威尔用一个反面的极限案例告诉了我们同一件事：团队的方向决定了协作的伦理价值。

这个对照在我脑子里挥之不去。

## 结语：从书页到屏幕

3000多天过去了，我把这两本书放回了书架。他们并排站在一起，一个封面是灰黑色的眼睛，一个封面是暖色调的少女。这个画面本身就像一个隐喻——在我二十岁的这一年，我同时看到了人性可以有多么冰冷，也可以有多么温柔。

我不是在说《1984》教会了我警惕技术，《小妇人》教会了我热爱生活。这种说法太简单了。实际上，是这两本书同时放在一起的时候所产生的化学反应，让我看到了一幅更完整的人性地图。在这幅地图上，电幕与炉火共存，新话与故事并存，恐惧与勇气并行。没有《1984》，我不会认识到技术的脆弱边界；没有《小妇人》，我可能不会相信在这些边界之外还有温暖的出口。

这段时间，我完成了课设——一个帮助老年人使用智能手机的简易教学应用。代码写得不算好，功能也很基础，但在写这个项目的过程中，我发现自己不再只考虑"性能"和"效率"了。我会想：这个按钮够大吗？老人的手指可能不太灵活；这个字体够清晰吗？也许有人老花了；这个提示音够明显吗？也许有人听力不太好。

这些念头，不是我之前写代码时会出现。我想，这就是那个寒假留在我心里的东西——被电幕冷光照过之后，我更懂得炉火为什么值得珍惜。

世界上所有的代码最后都要服务于人。而人是复杂的，有温斯顿的恐惧，也有乔的勇敢。读过这两本书之后，我似乎更清楚地看到了这一点。书不会告诉我该怎么写下一行代码，书会告诉我，在写下那行代码之前，应该先去想清楚——这行代码，是在砌墙，还是在生火。

我选择生火。
