# 文档结构与提纲规范 (Structure Blueprint)

## 3.1 元结构原则
采用"双线交织+螺旋上升"结构：
- 一条线向下：潜入《1984》的黑暗深渊，见证人性如何被制度碾碎
- 一条线向上：攀向《小妇人》的光明高地，感受人性如何在爱中绽放
- 交汇点：在每一次对比中，回归一个软件工程学子的自我叩问

## 3.2 通用章节骨架
```text
[标题区]
  └─ 标题公式：双书共读 + 人性两极 + 专业叩问

[序章/导语]（占全文5-8%）
  └─ 三要素：阅读的契机（为何同时读这两本书）+ 直观感受（一个让人窒息，一个让人疗愈）+ 核心问题（人性的边界在哪里？）

[主体论证区：四幕对比递进四重奏]（占全文65-75%）

  第一乐章：冰与火之歌——《1984》的极权深渊（500字）
    └─ 电幕、新话、思想警察——当技术成为枷锁
    └─ 温斯顿的恐惧：一个人如何在系统中失去自己
    └─ 意象锚点：灰暗的伦敦、无处不在的眼睛

  第二乐章：光与暖之诗——《小妇人》的人性花园（500字）
    └─ 马奇家的圣诞夜：贫穷却富有的灵魂
    └─ 乔的写作与成长：一个人在爱中成为自己
    └─ 意象锚点：温暖的炉火、阁楼上的笔和纸

  第三乐章：技术的两副面孔——从监控到关怀（600字）
    └─ 电幕vs.书信：沟通的两种本质
    └─ 新话vs.故事：语言是被操控还是被创造
    └─ 作为一名软件工程学生的冰冷发现：《1984》的噩梦正在某些角落悄然成真
    └─ 作为一名软件工程学生的温暖希望：技术也可以是马奇家的炉火

  第四乐章：在代码与人性之间——我的选择题（600字）
    └─ 在学习算法时，我想到的是优化效率还是优化人？
    └─ 数据隐私的困境：温斯顿日记与数字足迹
    └─ 团队协作中的温度：从马奇家的相处看敏捷开发

[结语/升华]（占全文10%）
  └─ 不引入新论据，将四重奏的主旋律融合
  └─ 回到阅读的本源：书不是为了逃避世界，而是为了更好地回到世界
```

## 3.3 段落微观结构
- **起承转合标记**：段与段之间使用意象过渡（如"从电幕的冷光转到炉火的暖光"）
- **信息密度**：对比段落不超过3层嵌套，否则会削弱冲击力
- **过渡规则**：从黑暗转向光明时，必须有一个"呼吸段"（单句或排比句作为过渡缓冲）

## 3.4 长度控制公式
- 总字数：3000字以上
- 各部分占比：导语8%、冰与火15%、光与暖15%、技术双面20%、专业选择20%、个人成长12%、结语10%

## 3.5 特殊要求
- 每一段读完《1984》后紧接着一段《小妇人》的片段作为"情感解药"
- 文中必须出现至少3处"我在敲这段文字的时候在想……"的真实体验表达
