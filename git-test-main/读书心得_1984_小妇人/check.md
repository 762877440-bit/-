# 文档质量检查规范 (Quality Checklist)

## 4.1 三级检查体系

### 一级：完整性检查 (Completeness)
- [ ] 标题是否包含"1984"、"小妇人"、"人性"等核心关键词？
- [ ] 导语是否交代了为什么同时读这两本书？
- [ ] 对两部作品的解读是否各有至少2个具体细节支撑？
- [ ] 结论是否回扣了导语中提出的核心问题？
- [ ] 是否从软件工程专业视角出发，而非仅仅是文学评论？

### 二级：一致性检查 (Consistency)
- [ ] 对《1984》的基调是否保持冷峻克制？没有突然变成温暖的？
- [ ] 对《小妇人》的基调是否保持温柔明亮？没有突然阴沉下去？
- [ ] 人称是否始终以第一人称"我"为主？
- [ ] 语言风格是否保持了"艺术加工"的水准？（修辞密度符合35%设定）
- [ ] 标点规范（中文正文用全角标点，引号内引语用全角）

### 三级：有效性检查 (Effectiveness)
- [ ] 对比测试：去掉所有《小妇人》的内容，文章是否只剩压抑？如是，则还需平衡
- [ ] 专业视角测试：换成一个中文系学生来写，内容是否相同？如果相同，则缺乏专业特色
- [ ] 呼吸感测试：是否连续超过500字没有"停顿点"？
- [ ] 时间胶囊测试：三年后重读，是否仍能感受到当时阅读的情感冲击？
- [ ] 行动导向测试：读者读完后是否能带走一个关于技术伦理的具体思考？

## 4.2 读书心得特化检查项
- [ ] 原文引用是否简明有度？（单次引用不超过50字）
- [ ] 个人思考是否依附于具体文本片段？（不空谈感受）
- [ ] 两部作品的对比是否自然，而非"虽然…但是…"的机械拼接？
- [ ] 语言是否达到了"艺术加工"的要求？（有至少2处让人印象深刻的修辞或意象）

## 4.3 评分与迭代机制
- **基线分**：满分100，一级检查每项12分，二级每项8分，三级每项4分
- **通过标准**：≥85分可输出；70-85分需标注待改进项；<70分重写
- **迭代指令**：若未通过，使用"基于[具体检查项]的反馈，请修正[具体位置]，要求[具体标准]"

## 4.4 常见失败模式
| 模式名称 | 症状 | 修复策略 |
|---------|------|---------|
| 失衡天秤倾斜 | 一部作品着墨过多，另一部沦为配角 | 强制统计两部作品的字数差，控制在10%以内 |
| 概念通货膨胀 | 堆砌意象无实质内容 | 每个意象后面必须跟一个"之所以选择这个意象，是因为..."的解释 |
| 情感透支 | 形容词密度>25% | 将所有"极其/无比/深深的"替换为具体场面描写 |
| 两张皮现象 | 文学分析与专业反思脱节 | 检查每段是否都有"技术连接句" |
| 音乐厅效应 | 过于优雅的抒情却没有个人真实感 | 检查是否出现了至少3处"我在敲这段文字的时候在想..."的真实体验 |
