# 主题锚定规范 (Topic Anchoring)

## 1.1 核心主题声明
- **主命题**：从《1984》的极权监控与《小妇人》的温暖成长中，探讨人性的两极——极端与美好，反思一名软件工程专业学生在技术伦理与人文关怀之间的价值选择
- **边界范围**：
  - 必须涵盖：《1984》中的极权主义、思想监控、语言操控；《小妇人》中的家庭温情、女性成长、独立人格、爱与牺牲
  - 明确排除：乔治·奥威尔生平考证、路易莎·奥尔科特的创作历程、纯粹的文学修辞分析、与主题无关的政治评论
  - 交叉地带：技术对人性的异化vs.技术对人性的解放、集体极权vs.个体自由、责任与爱的约束vs.爱的滋养
- **目标读者**：高校师生、关注技术伦理与人文素养的软件工程从业者

## 1.2 偏题检测机制
生成过程中，每完成一个章节需自检：
- [ ] 本节内容是否同时关联两部作品的核心命题？
- [ ] 是否在"人性极端"与"人性美好"之间是否建立了有效的对话？
- [ ] 是否体现了软件工程学子的独特视角与专业反思？

## 1.3 题材适配示例
| 题材类型 | 主命题示例 | 边界排除示例 |
|---------|-----------|------------|
| 对比分析 | 《1984》的"老大哥"与《小妇人》的马奇家——监控与关爱的一体两面 | 不展开大洋国经济体系、不比较两部作品的文学史地位 |
| 专业结合 | 算法监控时代的隐私困境：从《1984》看技术伦理的底线 | 不深入讲解具体加密算法实现 |
| 人性探讨 | 在温斯顿的恐惧与乔的勇敢之间：软件工程师的精神坐标 | 不罗列所有角色关系图谱 |

## 1.4 纠偏触发词
当内容偏离时，使用以下句式强制回归：
"回到核心命题：在《1984》的黑暗与《小妇人》的光明之间，我们需要聚焦的是人性如何在技术的浪潮中保持温度..."
