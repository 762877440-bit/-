# 项目规格说明书 (Specification)

## 1. 项目概述
- **项目名称**：TerminalAI - 智能终端助手
- **项目目标**：开发一个运行在终端界面的通用型 AI 对话 Agent，具备长期记忆、实时网页搜索和自主规划执行能力，面向普通消费者，使用中文交互，语气专业。
- **核心模型**：Qwen3.4-4b
- **交互方式**：纯文本，流式输出

## 2. 功能规格

### 2.1 核心对话功能
| ID | 功能 | 描述 |
|----|------|------|
| F-01 | 多轮对话 | 支持连续对话，上下文窗口至少 8k tokens |
| F-02 | 流式输出 | 逐字/逐词显示模型回复，提供实时反馈 |
| F-03 | 中文优先 | 默认使用中文交互，支持中英文混合输入 |
| F-04 | 专业语气 | 回复风格客观、准确、有条理，避免情绪化表达 |

### 2.2 长期记忆功能
| ID | 功能 | 描述 |
|----|------|------|
| F-05 | 跨会话记忆 | 用户再次启动终端时，能够回忆此前对话中的关键信息（如姓名、偏好、历史问题） |
| F-06 | 记忆自动管理 | Agent 自主判断哪些信息值得长期存储，无需用户手动标记 |
| F-07 | 记忆检索 | 每轮对话开始时，自动检索相关记忆并注入上下文 |
| F-08 | 记忆遗忘/更新 | 支持覆盖过时信息，避免记忆冲突 |

### 2.3 实时网页搜索
| ID | 功能 | 描述 |
|----|------|------|
| F-09 | 搜索触发 | Agent 自主判断是否需要联网获取最新信息（如新闻、事实查询、实时数据） |
| F-10 | 搜索执行 | 调用外部搜索 API（如 DuckDuckGo、Google、Bing）获取网页摘要或全文 |
| F-11 | 结果整合 | 将搜索结果作为上下文，辅助模型生成准确回答 |
| F-12 | 来源引用 | 在回复中注明信息来源（URL 或站点名称） |

### 2.4 自主规划与执行 (Agentic Behavior)
| ID | 功能 | 描述 |
|----|------|------|
| F-13 | 任务分解 | 对于复杂问题（如“比较昨天和今天的油价并给出省钱建议”），Agent 自行分解为多个子任务 |
| F-14 | 工具调用 | 自主决定调用搜索、计算、记忆读写等工具，并按照顺序执行 |
| F-15 | 错误恢复 | 当某个工具调用失败时，Agent 应尝试替代方案或向用户请求澄清 |
| F-16 | 执行日志 | 记录决策过程和工具调用链，便于调试（可通过 verbose 模式查看） |

### 2.5 终端交互体验
| ID | 功能 | 描述 |
|----|------|------|
| F-17 | 命令行启动 | 通过 `python main.py` 或自定义别名启动 |
| F-18 | 命令控制 | 支持 `exit`、`/clear`（清屏）、`/memory`（查看记忆）、`/search on/off` 等特殊命令 |
| F-19 | 用户提示符 | 显示用户名或时间戳，如 `You > `、`Agent > ` |
| F-20 | 彩色输出 | 区分用户输入、Agent 思考过程、最终回答、错误信息（使用 ANSI color） |

## 3. 技术规格

### 3.1 架构与框架
| 组件 | 技术选型 | 说明 |
|------|----------|------|
| 编排层 | LangGraph (Python) | 管理 Agent 工作流、状态、工具调用决策 |
| 模型服务 | Qwen3.4-4b (通过 Ollama 或 vLLM) | 本地部署，兼容 OpenAI API 格式 |
| 长期记忆 | Mem0 + 向量数据库 (Chroma / Milvus Lite) | Mem0 管理记忆的存取、摘要；向量库用于语义检索 |
| 网页搜索 | DuckDuckGo Search API (免费) 或 Tavily | 优先使用无 API Key 方案 |
| 流式输出 | asyncio + 生成器 | 支持逐 token 输出 |
| 终端界面 | Rich 或 prompt_toolkit | 增强交互体验 |

### 3.2 硬件与环境要求
| 项目 | 最低配置 | 推荐配置 |
|------|----------|----------|
| CPU | 4 核 | 8 核 |
| RAM | 8 GB | 16 GB |
| GPU | 无（CPU 推理，速度较慢） | NVIDIA GPU 6GB 显存（支持 4-bit 量化） |
| 存储 | 20 GB | 40 GB |
| 操作系统 | Linux / macOS / Windows WSL2 | Linux Ubuntu 22.04 |
| Python 版本 | 3.10+ | 3.11 |

### 3.3 数据与存储
| 数据类型 | 存储方案 | 生命周期 |
|----------|----------|----------|
| 对话历史（短期） | 内存中的列表，用于当前会话上下文 | 会话结束时清空 |
| 长期记忆 | 向量数据库 + JSON 索引文件（会话外持久化） | 永久，直到用户手动删除 |
| 工具调用日志 | 可选写入本地日志文件（`.logs/`） | 可配置保留 7 天 |

### 3.4 性能规格
| 指标 | 目标值 | 备注 |
|------|--------|------|
| 首 token 延迟 | < 2 秒 | 从用户输入到第一个字符输出 |
| 生成速度 | 20-30 token/秒 | 在推荐 GPU 上 |
| 记忆检索延迟 | < 500 ms | 向量相似度搜索 |
| 搜索超时 | 5 秒 | 超过则降级为无搜索回答 |
| 并发用户 | 1（单用户终端） | MVP 阶段不要求多用户 |

## 4. 非功能规格

### 4.1 可维护性
- 所有工具（搜索、记忆、模型调用）以插件形式实现，便于替换
- 配置文件 (`config.yaml`) 管理 API 密钥、模型参数、路径
- 使用 Python logging 分级记录运行信息

### 4.2 可测试性
- 单元测试覆盖核心模块（工作流图节点、记忆接口、工具函数）
- 提供 Mock 模型响应和 Mock 搜索引擎用于 CI 测试

### 4.3 安全性
- 本地运行，不主动上传用户数据
- 搜索请求通过代理可选，避免 IP 封锁
- 敏感命令（如删除记忆）需二次确认

### 4.4 可扩展性
- 未来可轻松增加新工具（如计算器、天气预报 API、本地文件读取）
- 支持切换其他模型（只需修改 API endpoint）

## 5. 交付物与里程碑
| 阶段 | 里程碑 | 产出 |
|------|--------|------|
| M1（基础对话） | 终端能调用 Qwen 流式回复 | `main.py`，无记忆、无搜索 |
| M2（长期记忆） | 集成 Mem0，跨会话记用户信息 | 记忆模块，测试对话 |
| M3（搜索集成） | Agent 可自主搜索并整合结果 | 搜索工具 + 决策节点 |
| M4（完整 Agent） | 复杂任务分解与工具链执行 | LangGraph 完整工作流 |
| M5（优化与交付） | 性能调优、文档完善 | 最终代码 + 用户手册 |

## 6. 约束与假设
- 假设用户拥有可运行 Qwen3.4-4b 的硬件（或允许使用第三方 API 作为备选）
- 假设网络环境允许访问搜索引擎（国内需考虑防火墙）
- 假设用户知情并同意本地存储对话记忆
- 暂不考虑多用户、分布式部署
